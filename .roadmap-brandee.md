# YAY! (Accessible Yahtzee Scorecard) - Development Roadmap

## Phase 1: Foundation (v1.0.0)
- Establish single-file HTML/JS/CSS architecture.
- Create strict ARIA focus trap locking the user into the `#game-container`.
- Implement safe initialization button to bypass browser auto-focus blocks.
- Define the `window.YAHTZEE_STATE` data object.

## Phase 2: Navigation (v1.0.1)
- Build dynamic HTML table for sighted players to track the screen reader.
- Wire Up/Down arrow keys for sequential virtual cursor movement.
- Implement array-wrapping to loop from top to bottom continuously.
- Route movement to `aria-live` announcements (e.g., "Aces. Empty.").

## Phase 3: Scoring Engine (v1.0.2)
- Implement `Enter` key toggle between Navigation Mode and Score Mode.
- Build "Stepper" logic (Up/Down arrows) to adjust scores based on category rules.
- Add `Shift` modifier for 5x fast-scrolling.
- Build dynamic math engine (Upper Subtotal, Bonus, Lower Total, Grand Total).

## Phase 4: Advanced Navigation (v1.0.4 - v1.0.5)
- Map hotkeys for instant jumps (1-6 for Upper, mnemonic letters for Lower).
- Map `PageUp` / `PageDown` for "Express Lane" jumping between Subtotals/Totals.
- Implement Yahtzee Bonus locking logic (unavailable until Yahtzee = 50).

## Phase 5: Memory & Polish (Target: v1.1.2)
- Implement `localStorage` hydration to save state on every entry.
- Build Game History (`E` key) to announce last 5 saved scores with timestamps.
- Build Leaderboard (`0` key) to announce top 10 all-time scores with timestamps.
- Build safe Reset Workflow (`Q` key) with confirmation prompt.
- Implement Developer Shortcuts (`Alt+Shift` or physical `e.code`) for rapid math testing.
